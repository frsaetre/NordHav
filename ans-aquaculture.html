AQU-001
Developed Mandatory

Long-Term Production Plan

Requirement: Support long-term (3–5 year) production planning incorporating: smolt release schedules, growth projections, harvest forecasts, site rotation/fallowing, and MAB (Maximum Allowed Biomass) capacity constraints.
Summary: Delivered via AquaMonitor MDA (Gap G-01) — purpose-built long-term production planner on Power Platform.
Show detailed solution response

Long-term production planning is one of the core capabilities delivered by AquaMonitor, the custom Model-Driven App (MDA) built on Microsoft Power Platform and Dataverse (Gap G-01). This is not a standard D365 F&O feature because aquaculture production planning has unique requirements that differ fundamentally from manufacturing planning:

  • Planning horizon: 3–5 year rolling plan covering the full salmon lifecycle from smolt release to harvest
  • Smolt release scheduling: plan stocking events by site, pen, and generation with timing based on smolt production capacity, MAB availability, and seasonal considerations
  • Growth projections: thermal growth coefficient (TGC) models combined with historical temperature profiles predict biomass growth by pen/site/generation over the planning horizon
  • Harvest forecasting: based on growth projections and target harvest weights, the plan predicts harvest volumes by week/month, feeding into processing plant capacity planning and sales forecasting
  • Site rotation & fallowing: production plan respects mandatory fallowing periods between production cycles, coordinating with zone-level agreements
  • MAB capacity constraints: the planner enforces Maximum Allowed Biomass limits per license and nationally, preventing over-stocking in the plan

AquaMonitor integrates bidirectionally with D365: harvest forecasts feed into D365's master planning for processing and sales, while D365 financial data (budgets, costs) informs production economics in the planner. The planning engine uses Power Automate for automated schedule generation and scenario comparison.

AQU-004
Developed Mandatory

MAB Monitoring

Requirement: Real-time monitoring of standing biomass against Maximum Allowed Biomass (MAB) limits per license and total. Alerts when approaching threshold (e.g., 90%, 95% of MAB).
Summary: Delivered via AquaMonitor MDA — real-time MAB monitoring with configurable threshold alerts (90%, 95%).
Show detailed solution response

AquaMonitor provides real-time MAB monitoring as a core dashboard feature:

  • Biomass calculation: daily standing biomass estimated per pen (fish count × estimated average weight based on growth model), aggregated to site, license, and company total
  • MAB comparison: standing biomass displayed as a percentage of MAB limit per license, with visual gauge charts showing utilization
  • Threshold alerts: Power Automate triggers notifications when biomass approaches MAB limits:
    • 80% — Informational (planning team notification)
    • 90% — Warning (operations manager + biology team notification)
    • 95% — Critical (management team + production director notification, harvest acceleration recommended)
    • 100% — Breach (executive alert, regulatory non-compliance risk)
  • Forecast projection: the dashboard shows not just current biomass but projected biomass based on growth models, alerting if MAB limits will be reached before planned harvest dates

This feature is critical for regulatory compliance — exceeding MAB limits can result in fines and license sanctions. The predictive element ensures NordHav can adjust harvest plans proactively rather than react to limit breaches.

Biomass Estimation

Requirement: Maintain daily biomass estimates per pen using: last known fish count, cumulative mortality, estimated growth (TGC/SGR models), and periodic calibration from sampling/weighing.
Summary: Delivered via AquaMonitor MDA — daily biomass estimation using growth models calibrated by periodic sampling.
Show detailed solution response

AquaMonitor maintains daily biomass estimates per pen:

  • Calculation: Biomass = estimated fish count × estimated average weight. Fish count = last verified count − cumulative mortality. Average weight = last sampled weight, grown forward daily using the growth model.
  • Growth model: Thermal Growth Coefficient (TGC) model uses: last known weight, daily water temperature, and a species/strain-specific TGC parameter to predict daily weight gain
  • Calibration: biomass estimates are periodically recalibrated when sample weighing events occur (AQU-029). The model adjusts to match actual weights and continues projecting forward.
  • Confidence tracking: each biomass estimate carries a confidence indicator based on: days since last calibration, variance between modeled and actual at last calibration, and mortality uncertainty

Biomass estimation is the foundation for MAB monitoring, feed budgeting, harvest planning, and financial valuation. The model accuracy is typically within 5-10% of actual biomass when calibrated monthly.

Growth Modeling

Requirement: Calculate growth metrics: Thermal Growth Coefficient (TGC), Specific Growth Rate (SGR), Condition Factor (K-factor). Support growth model tuning based on actual vs. predicted weight.
Summary: Delivered via AquaMonitor MDA — TGC, SGR, and K-factor calculations with model tuning capability.
Show detailed solution response

AquaMonitor calculates key growth metrics:

  • TGC (Thermal Growth Coefficient): TGC = (W₂^(1/3) − W₁^(1/3)) / (Σ°C × days) — used for growth projections
  • SGR (Specific Growth Rate): SGR = (ln(W₂) − ln(W₁)) / days × 100 — daily growth rate percentage
  • K-factor (Condition Factor): K = (W / L³) × 100 — body condition index from weight and length sampling
  • Model tuning: when actual weights from sampling diverge from predicted, site biologists can adjust the TGC parameter for each pen to improve future predictions. Historical TGC performance is tracked for model refinement.

These metrics are displayed on pen and site dashboards, used for growth prediction in production planning, and reported to management as operational KPIs. Cross-site comparison of growth parameters helps identify best-performing sites and conditions.
